rosbridge实战（一）

websocket通信

WebSocket 是一个基于Web的协议，旨在使client应用程序和远程主机之间进行双向通信。如果最初的握手成功，则两个实体可以来回发送数据。WebSocket 协议使用ws: 和 wss: URL协议，以分别代表不安全和安全的 WebSocket 请求。基于 WebSocket 的解决方案包括 client 和 server。本地 client 对于 WebSocket 的支持在大多数浏览器中可用，服务器端支持WebSocket使其成为一个实现双向通信的完整解决方案。任何支持原生 WebSocket 的浏览器应该可以使用 HTML5 WebSocket API 来运行一个简单的 HTML 和基于 JavaScript 的 client 应用程序。

rosbridge

平时控制机器人是用SSH直接在工控机上进行的，但实际中需要一个上位机环境，比如一个windows电脑、iOS手机、Android手机，这就引出了一个问题：如何让不同的平台与ROS通信？ROS官方提供了一个用于非ros系统和ros系统进行交互通信的功能包rosbridge。

主要包含两个部分，协议和实现。其中Protocol部分提供了非ros系统和ros系统通信的具体格式，包括话题的订阅，消息的发布，服务的调用，参数的设置和获取，图片信息的传递等等，都是JSON格式的字符串。 Implementation部分是rosbridge的具体实现，包含rosapi，rosbridge_library,rosbridge_server等包。rosapi通过服务调用使某些ROS action可访问，包括获取和设置参数，获取主题列表等;rosbridge_library是核心rosbridge包，rosbridge_library负责获取JSON字符串并将命令发送到ros，反之亦然；rosbridge_server就提供了websocket的服务端，负责通信的传输层，包括websocket，tcp，udp等几种格式。

rosbridge针对BS架构和CS架构分别提供了相应的解决方案。

  • 在BS架构中，rosbridge提供了roslibjs库，roslibjs是一个通过浏览器和ros系统进行交互的javascript库，通信方式为websocket，在终端通过如下代码roslaunch rosbridge_server rosbridge_websocket.launch
    启动websocket，并在html中载入roslibjs库，就可以在html中依照指定的格式和ros系统进行通信了。

  • 在CS架构中，rosbridge提供了TCP协议和UDP协议两种方式，其中因为TCP协议需要三次握手，因此比较稳定准确，而UDP协议速度更快，但会发生丢包，因此存在不准确性。相应的启动方式分别为roslaunch rosbridge_server rosbridge_tcp.launch 和roslaunch rosbridge_server rosbridge_udp.launch 具体的通信内容都遵从JSON格式的字符串

我们常用的还是rosbridge_websocket.launch，默认接口是9090，IP为当前机器的。所有机器必须在同一个网段，rosbridge利用JSON API的方式让不能装ROS的平台也能使用ROS，通过rosbridge启动websocket server，另一平台使用浏览器或桌面客户端，实现二者通信。

按照rosbridge_suite官方教程安装好rosbridge，按照例程一步步来，保存为test.html之后，先启动roscore，然后roslaunch rosbridge_server rosbridge_websocket.launch，执行rostopic echo /cmd_vel准备查看话题的消息，用浏览器直接打开html文件，此时会看到话题消息中twist的情况。
